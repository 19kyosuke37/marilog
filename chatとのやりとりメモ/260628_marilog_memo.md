## 🎯 一言で言うと

**「自分と似た条件の先輩が、結婚してどうなったかを検索できるサービス」**

結婚に悩む若者向けに、リアルな先輩データを構造化して届けるプラットフォーム。
既婚ユーザーが経験をデータとして登録し、結婚前ユーザーが参考にできる。

※このサービスはKyo自身が当事者（5年交際・結婚に踏み出せない・お金が不安）として
「こういうものが欲しかった」から生まれた。この当事者性は拡散の最大の資産。

## 🏗️ 技術スタック

- **フレームワーク**：Next.js 16 (App Router) + TypeScript
- **バックエンド**：Supabase（Auth / PostgreSQL / RLS）
- **スタイリング**：Tailwind v4（`@import "tailwindcss"`）+ CSSカスタムプロパティ
- **認証**：Supabase Auth（メール/パスワード + Google OAuth）、`@supabase/ssr`でSSRセッション管理

### 開発コマンド
```bash
npm run dev    # 開発サーバー起動（localhost:3000）
npm run build  # ビルド
npm run lint   # ESLint
```

## 🗄️ DBテーブル構成

- `profiles` — 全ユーザー共通（`user_type: married/before`、`avatar_url`含む）
- `senpai_profiles` — 既婚ユーザーの詳細データ（`profiles`と1:1）
- `posts` — 投稿（`post_type: married/before`）
- `comments` — コメント
- `comment_likes` — コメントいいね
- `notifications` — コメント通知（DBトリガーで自動生成）

※詳細スキーマは `supabase/schema.sql` および `SPEC.md` 参照

## 📣 拡散戦略（骨格・合意済み）

### 基本方針
- **既婚者を先に集める**（語る側がいないと成立しないため）
- 動機は「他人のリアルな家計が**覗ける**」＋「匿名で本音を**吐き出せる**」

### Xアカウント運用
- **ゼロ円・朝昼夜の小分け運用**（平日は小分け、週末にまとめてコンテンツ仕込み）
- キャラ＝**「結婚に踏み出せない当事者」（Kyo自身）**
- 二段構え：
  - 横（同志）：「同志はいるか？どう考えてる？」→ 結婚前の仲間＝母数拡大
  - 縦（先輩）：「先輩教えて〜」→ 既婚者の本音＝語る側を確保
- 集まったリプ＝MARILOGに必要なリアルデータそのもの

### 🔑 公開の順番（重要）
- **当面MARILOGは伏せる**（まだ人に見せられる完成度ではないとKyoが判断）
- Xを「公開前のユーザーインタビューの場」として使い、集まった声で磨く
- Kyoが「見せられる」と思えたらプロフィールにリンク設置

### フェーズ
- **1（今〜）**：MARILOG伏せたままX発信。声を集めながら磨く
- **2**：プロフィールに静かにMARILOG設置。仲間に最初に見せる
- **3**：本格的に流入を作る
